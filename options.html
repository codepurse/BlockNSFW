Welcome

What’s new in BlockNSFW, and a note from the developer.

What’s New

  • AI Text Protection (multilingual) — A new on-device AI reads the page's text to catch adult sites the keyword and blocklist filters miss — especially non-English ones (Russian, Chinese, Korean, Thai, Arabic, and more). It blocks pages it's confident about, and teams up with the image AI to catch borderline ones. Nothing leaves your device.
  • AI Image Protection — A new on-device AI looks at the actual picture to block adult images, even when nothing in the web address gives them away. Everything stays on your device — your images are never uploaded.
  • Instant blocking — Flagged images are now hidden the moment they're checked, so adult content no longer flashes on screen before it's blocked.
  • IDN / Punycode Coverage — Internationalized adult mirrors with xn-- hostnames now load and block as designed.
  • Non-English Hostname Filter — Adult-hostword coverage for Chinese, Korean, Russian, Arabic, and Thai hostnames (e.g. 色情.com, 야동.com, порно.org).
  • Smarter Bundled Blocklist — Regenerated and synced with the runtime parser, so offline installs get the same coverage.

How it works

Both models are bundled with the extension and run on your device. The image model runs in a dedicated Web Worker; the ~150 KB text model scores the page in milliseconds. Whole-page text blocking uses a conservative threshold, with a lower threshold when the image blocker also flags content on the page.

DNS Protection New

Block adult domains at the DNS level using Cloudflare for Families. This adds an extra layer of protection by checking domains against a constantly-updated global filter before the page even loads.

Enable DNS Filtering Query Cloudflare's family-safe DNS to detect and block adult domains automatically
🌐 DNS: Off
How it works

When enabled, each new domain you visit is checked against Cloudflare's family-safe DNS (1.1.1.3) via DNS-over-HTTPS. Results are cached locally so repeat visits have zero delay. This uses a free public service — no account or API key required.

Disable Instagram Reels Hide reel entry points and redirect direct reel pages back to the main Instagram feed
🔎 Safe Search: Off 📵 FB Reels: Off 📵 IG Reels: Off
How it works

Uses Chrome's declarativeNetRequest engine to rewrite search URLs (safe=active, adlt=strict, etc.) before the request leaves your browser. YouTube 18+ videos, Facebook Reels and Instagram Reels are blocked locally with an early page redirect plus DOM filtering. Nothing is sent to a third party.

For cookie-backed engines (Qwant, AOL Search, Presearch), BlockNSFW additionally pins the SafeSearch cookie to strict on every response and silently redirects the engine's own settings page back to its home, so the toggle cannot be flipped off from the UI.
